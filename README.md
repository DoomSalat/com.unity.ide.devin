# com.unity.ide.devin

Unity Editor integration for [Devin IDE](https://devin.ai).

Registers Devin as an external script editor in Unity — auto-discovery, file opening with line navigation, and `.csproj` / `.sln` generation for IntelliSense via OmniSharp.

---

## Two variants

This repository provides two independent implementations on separate branches. Choose the one that fits your setup.

### `master` — Self-contained generation

Generates `.csproj` and `.sln` files from scratch using Unity's `CompilationPipeline` API. No other editor plugin required.

**Use when:** You have no Rider or Visual Studio installed, or you want full control over what gets generated.

**Trade-offs:**
- Independent of other editor plugins — works out of the box
- Configurable per package type: Embedded, Local, Git, Registry, Built-in, Player assemblies
- Requires .NET SDK 6.0+ on the machine (OmniSharp uses it to evaluate `.csproj` files)

**Installation:**
```json
"com.unity.ide.devin": "https://github.com/DoomSalat/com.unity.ide.devin.git"
```

---

### `via-reflection` — Delegate to installed editor *(faster)*

Delegates `.csproj` / `.sln` generation to an already-installed editor plugin — Rider or Visual Studio Tools — via Unity's `IExternalCodeEditor` API and internal reflection. Devin acts as a thin proxy that triggers the other editor's generator and then writes `omnisharp.json` and `Directory.Build.props` on top.

**Use when:** You already have `com.unity.ide.rider` or `com.unity.ide.visualstudio` in your project and want IntelliSense files generated by the same engine those tools use internally.

**Why it's faster:**
- Rider's and VS Tools' generators are heavily optimized and cache assembly data between runs
- Incremental sync (`SyncIfNeeded`) is handled by the delegate — only changed assemblies are re-evaluated
- VS Tools' internal `ProjectGenerator.Sync()` is called directly via reflection, bypassing the current-editor guard that would otherwise block it when Devin is selected

**Trade-offs:**
- Requires `com.unity.ide.rider` or `com.unity.ide.visualstudio` present in `Packages/manifest.json`
- Generated `.csproj` style depends on the delegate (Rider → SDK-style, VS Tools → old-style or SDK depending on version)
- Uses internal reflection — may break if Unity changes editor plugin internals between versions

**Installation:**
```json
"com.unity.ide.devin": "https://github.com/DoomSalat/com.unity.ide.devin.git#via-reflection"
```

**GUI:** The Preferences panel shows a dropdown to pick which registered editor to delegate to, the installation status of the selected editor, and the live state of `.sln`, `omnisharp.json`, and `Directory.Build.props`.

## Common requirements

- Unity 2021.3+
- Devin IDE installed at:
  - **Windows:** `%LOCALAPPDATA%\Programs\Devin\Devin.exe`
  - **macOS:** `/Applications/Devin.app`
  - **Linux:** `/usr/bin/devin` or `/usr/local/bin/devin`

`master` additionally requires .NET SDK 6.0+ (OmniSharp uses it to load SDK-style `.csproj` files — Visual Studio is not required).

`via-reflection` delegates requirements to the chosen editor plugin (e.g. Rider installed locally, or Visual Studio installed for VS Tools).

---

## Installation (local package)

```json
"com.unity.ide.devin": "file:/path/to/com.unity.ide.devin"
```

## Setup

1. Open **Edit → Preferences → External Tools**
2. Set **External Script Editor** to **Devin**
3. *(master only)* Configure which package types to generate `.csproj` for
4. *(via-reflection only)* Select the delegate editor from the dropdown
5. Click **Regenerate project files**

OmniSharp will pick up the solution automatically via the generated `omnisharp.json`.

## OmniSharp notes

Both variants write two support files after every sync:

**`omnisharp.json`** — points OmniSharp at the correct `.sln`:
```json
{
  "msbuild": {
    "solution": "D:\\MyProject\\MyProject.sln",
    "loadProjectsOnDemand": false
  }
}
```

**`Directory.Build.props`** — prevents MSBuild from resolving .NET Framework targeting packs (required when Visual Studio is not installed):
```xml
<Project>
  <PropertyGroup>
    <SkipGetTargetFrameworkProperties>true</SkipGetTargetFrameworkProperties>
    <NoRestore>true</NoRestore>
  </PropertyGroup>
</Project>
```

`master` generates SDK-style projects (`<Project Sdk="Microsoft.NET.Sdk">` with `<TargetFramework>net471</TargetFramework>`). Assembly references not included in generation are resolved as `HintPath` entries pointing to `Library/ScriptAssemblies/`.

## Features

- **Auto-detects Devin** on Windows, macOS, and Linux
- **Opens files** at the correct line and column via `--goto`
- **Filters file types** — non-code assets (prefabs, materials, scenes) stay in Unity's own editors
- **`omnisharp.json`** auto-generated with the correct solution path
- **`Directory.Build.props`** auto-generated to keep MSBuild compatible with .NET Core SDK
